# AI-Assisted Contributions

We welcome contributions made with the help of AI tools — **if a human stands
behind them.** Choose / adapt the rules below for your project.

## Our expectations
1. **You read and understand every line you submit.** "The AI wrote it" is not
   a defense for code you can't explain.
2. **You ran it.** Code that was never executed will be closed.
3. **Tests included** for behavior changes.
4. **Disclose AI use** in the PR description (a checkbox below is fine). This is
   for context, not punishment.
5. **No bulk / drive-by PRs.** One thoughtful change beats ten generated ones.

## Not accepted
- Auto-generated PRs opened at scale without review.
- AI-generated "security reports" with no reproducible proof of concept.
- Reformatting / dependency-bump spam with no discussion.

> Tip: contributors can self-check before opening a PR with a local tool like
> `prcheck` (no API key, nothing leaves their machine).
